プロジェクト

Adobe Experience Manager（AEM）および自社開発CMSを用いた実績の一部を紹介します。 エンタープライズ向けコンテンツ運用、ローカライズQA、グローバル連携、 ならびにフレームワークレベルでの検証を中心に取り組みました。

AEM：コーポレート・製品・患者サポートページ

Adobe Experience Manager（AEM）を使用し、日本市場向けに コーポレート、製品、疾患、患者サポートページの作成および運用を担当しました。

対応ページ種別：

  • コーポレートページ
  • 製品ページ
  • 疾患情報ページ
  • 患者サポートページ

担当業務

  • AEM を用いた新規ページ作成および既存ページ更新
  • PSD デザインデータからの画像抽出およびレイアウト調整
  • ブラウザ／デバイス検証（Chrome、Edge、iPad、iPhone、Android）
  • 日本語ページに対する QA・ローカライズQA
  • 日本側関係者との要件・スケジュール調整
  • 海外 AEM チームとの連携による公開・本番リリース対応

リード対応および納品管理

AEM リードとして業務範囲を拡大し、2名のチームメンバーを担当。 複数サイトにまたがるページ進行管理と、 日本側クライアントおよびグローバルチームとの双方向コミュニケーションを行い、 安定したリリースとトラブルのない運用を実現しました。

イベントページ更新（Before / After）

Web セミナー等のイベントページにおいて、 公開前後のコンテンツ更新、レイアウト調整および QA を担当。 イベントスケジュールに合わせ、正確な情報提供と円滑な更新対応を行いました。

自社開発CMS：AEM から CMS への移行

AEM 上の製品・疾患・患者サポートページを、 自社開発 CMS へ移行し、日本市場向けに安定したリリースを実施しました。

対応ページ種別：

  • 製品ページ
  • 疾患情報ページ
  • 患者サポートページ

担当業務

  • AEM ページの CMS への移行作業
  • デザイン要件に基づくレイアウト調整
  • ブラウザ／デバイス検証
  • 移行後の日本語ローカライズQA
  • 本番環境へのページ公開
  • 日本側クライアントとの要件・スケジュール調整

フレームワーク検証・グローバル連携

新規導入された CMS フレームワークを初期段階から検証し、 挙動や制約を整理してチームへ共有。 海外チームとは英語でのミーティング、メール、チャットを通じて、 不具合報告、仕様確認、リリース課題の調整を行いました。 クライアント要望に基づくデザイン確認や、 実運用を踏まえた UI 改善提案も行いました。

積極的なフレームワーク調査と円滑なチーム間連携により、 移行プロジェクトはスムーズに進行し、良好な評価を得ました。

自社開発CMS：新規ページ制作・公開

自社開発 CMS を使用し、日本向けの製品・疾患・患者サポートページを新規構築し、 公開まで一連の対応を行いました。

対応ページ種別：

  • トップページ
  • ランディングページ
  • Web セミナーページ
  • 製品ページ
  • 疾患情報ページ
  • 患者サポートページ

担当業務

  • CMS を用いた新規ページ作成
  • 日本語ローカライズコンテンツの入力および検証
  • 日本語表記に配慮したレイアウト確認
  • 公開前の最終 QA
  • 単独での本番公開対応

ナレッジ共有

CMS フレームワークの挙動を整理し、 使用方法や既知の制限事項をチームへ共有。 CMS に関する問い合わせ窓口として対応し、 オンボーディング支援およびナレッジ定着に貢献しました。
